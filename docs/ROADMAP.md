# Roadmap

The roadmap separates concept proof from production commitments. Dates, partner commitments, and a real TERRAST connection remain intentionally unassigned until owners and contracts are confirmed.

## P0 — Concept MVP completion

Outcome: one coherent, operable synthetic demo with evidence-backed quality.

- Landing, demo login/role switch, company dashboard, Data Sync Center.
- Mock/CSV/JSON connector boundary, dry-run/diff/conflict/idempotent apply, provenance.
- Metric history, SSBJ/ISSB-oriented original requirement summaries, guided response.
- GHG/Scope 3, supplier request/response, risk/opportunity, transition plan.
- Deterministic/OpenAI-optional assistant with evidence IDs and structured validation.
- Review/return/revise/approve, append-only audit, operator aggregates/consent summary.
- Print/CSV/JSON reports and fictional marketplace.
- Multi-tenant RLS, private Storage, synthetic 3-company/3-year seed.
- Unit/integration/E2E/accessibility/security smoke tests and current documentation.
- Preview and Production deployment only when externally authorized and verified.

Exit evidence for the synthetic demo: verified URLs, golden-scenario browser result, all seven CI jobs green, no secret/real-data finding, and updated `AI_HANDOFF.md`. Live RLS/Storage negative tests are required before Supabase mode, not for the credential-free demo deployment.

## P1 — Controlled pilot readiness

Outcome: an approved, supportable pilot with tightly bounded data and users.

- TERRAST discovery and interface-control document; sandbox adapter and contract tests.
- Implement the non-AI `SupabaseRepository` schema adapter and audited server commands, then execute migration/RLS/Storage/pgTAP tests against an isolated remote project.
- Pilot tenancy/group model, onboarding/offboarding, MFA/SSO decision, role reviews.
- Approved SSBJ/ISSB catalog governance and authoritative emission-factor process.
- Consent/legal/privacy notices, DPA/subprocessors, retention/deletion/export/legal hold.
- Malware quarantine/scanning, distributed rate limiting/WAF, CSP/security headers.
- Purpose-built aggregate views with minimum cohorts, suppression and re-identification review.
- OpenAI/provider data controls or an approved alternative; tenant opt-out and redaction.
- Observability, SIEM/security alerts, backups/PITR, restore and incident exercises.
- Performance/load tests, accessibility audit, penetration test, threat-model refresh.
- Pilot support/SLA, change management, training, feedback, success measures, rollback.

Exit evidence: signed approvals, pilot runbook, sandbox/negative/load/security test reports, restore exercise, named operational owners, and go/no-go decision.

## P2 — Production and ecosystem scale

Outcome: resilient multi-party operation after pilot validation.

- Production TERRAST integration only under approved contract, data rights, SLA, and monitoring.
- Scalable framework/catalog updates and additional authorized imports/connectors.
- Group-company consolidation, advanced Scope 3 workflows, assurance workspaces.
- High-quality exports/data products with privacy budgets and downstream contracts.
- Enterprise SSO/SCIM, delegated administration, fine-grained evidence grants.
- Regional/residency strategy, HA/DR targets, capacity/cost controls, continuous control monitoring.
- Formal operating governance, audit/certification roadmap, vendor/third-party reviews.
- Internationalization and additional market workflows only after local MVP validation.

No P2 item implies JPX adoption, approval, financing, or a nationwide mandate. Those outcomes require separate stakeholder decisions and communications authorization.

## Deferred / explicitly not promised

- A fabricated production endpoint or inferred TERRAST schema.
- Unlicensed standards-text hosting.
- Automatic compliance/assurance determination.
- Real financial-product recommendations or real marketplace commercial terms.
- Unconsented company-detail access by a platform operator.
- Production deployment or partnership announcement without verified authority.
